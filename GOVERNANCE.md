# GOVERNANCE.md — 앳플리 AI 거버넌스 초안 (v0)

목적: AI를 "많이" 쓰는 게 아니라 "안전하게" 많이 쓰기 위한 최소 기준.
살아있는 초안이며, 사고/이슈/신규 에이전트가 생길 때마다 보완한다.

## 1. 데이터 등급
- 공개(Public): 제품/브랜드 공개 정보 -> AI 사용 자유
- 내부(Internal): 정책/매뉴얼/업무기준 -> 사내 시스템(사내 RAG) 내에서 사용
- 민감(Sensitive): 고객 개인정보(주문번호/연락처/주소), 직원 인사/평가/급여
  -> 마스킹 없이는 외부 LLM 입력 금지

## 2. 외부 LLM vs 내부 사용 기준
- 민감 데이터는 외부 API로 원문 전송 금지. 필요 시 비식별화/마스킹 후에만.
- 공개/내부(비민감) 데이터는 승인된 모델(예: Claude)로 사용 가능.

## 3. 사람 승인이 필요한 지점 (Human-in-the-loop)
- 고객에게 직접 나가는 응답(CS/VOC 답변)
- 환불/보상/계약 등 금전적·법적 영향이 있는 결정
- 대외 공개 콘텐츠(상세페이지/광고 문구)의 최종본
- 위 항목은 AI가 "초안"까지, 최종 판단·발송은 사람이 한다.

## 4. 마스킹 원칙
- 이름/전화/주소/주문번호/카드정보 등은 AI 입력 전에 가린다.
- 공개 챗봇 화면에는 개인정보 입력 금지 안내를 노출한다 (앳플리봇에 이미 적용).

## 5. 책임
- AI 출력은 "초안·참고"이며, 최종 책임은 이를 채택한 담당자/승인자에게 있다.
- 중요한 판단은 원문 문서와 담당자 확인을 우선한다.

## 6. 로그·기록
- 질문/답변/참조문서/사용모델을 기록한다 (개선·감사 목적).
- 민감정보는 로그에도 남기지 않거나 마스킹한다.
- 보관 기간은 회사 정책에 따른다 (초안: 별도 확정 필요).

## 7. 시크릿·접근
- API 키/비밀번호는 Key Vault에서만 관리. 코드/깃에 두지 않는다.
- 에이전트별 네임스페이스로 접근을 격리한다.

## 갱신
- 이 문서는 살아있는 초안(v0). 정기적으로 재검토하고 보완한다.
